```
H2: [Subtopic 2]
H3: [detail within subtopic]
H2: Frequently Asked Questions
H2: More From the Journal
```

### 2.4 Keyword Placement Rules

For each target page, place the primary keyword in:
1. Title tag (first 60 characters)
2. Meta description (first 110 characters)
3. H1 tag
4. First 100 words of body text
5. One H2 heading
6. Image alt text of the hero/featured image
7. URL slug

Do NOT stuff — target keyword density of 1–2% naturally. Use semantic variations (LSI keywords) throughout.

## Part 4 — Technical SEO

### 4.1 Core Web Vitals Targets

Google's Page Experience signals (LCP, INP, CLS) directly affect ranking, especially on mobile where 80%+ of Bangladesh traffic originates.

| Metric | Good | Needs Work | Target |
|---|---|---|---|
| LCP (Largest Contentful Paint) | < 2.5s | > 4.0s | < 2.0s |
| INP (Interaction to Next Paint) | < 200ms | > 500ms | < 150ms |
| CLS (Cumulative Layout Shift) | < 0.1 | > 0.25 | < 0.05 |

**LCP optimizations (currently at risk):**
- Ensure hero product images use `fetchpriority="high"` and are preloaded via `<link rel="preload">`
- Move hero image above the fold; avoid lazy-loading it
- Serve images from a CDN (Vercel's edge network covers Asia — already handled if deployed on Vercel)
- Use Next.js `<Image>` component with `priority` prop for all LCP candidates

**INP optimizations:**
- Avoid heavy JS on interaction paths (add to cart, filter, sort)
- Debounce filter/search inputs (100ms)
- Use `useTransition()` for filter state updates — prevents blocking renders
- Defer non-critical scripts (analytics, chat widgets) with `strategy="lazyOnload"`

**CLS optimizations:**
- All images MUST have explicit `width` and `height` attributes (or Next.js Image handles this automatically)
- Reserve space for font loading with `font-display: swap` or `optional`
- Avoid inserting content above existing content (banners, cookie notices)
- Product card grid: use explicit `aspect-ratio` on image containers

### 6.2 Content Formats AI Systems Prefer

Write content that answers questions directly in the first paragraph. AI systems extract the first clear declarative answer:

❌ Bad: "In this article we will explore the different options available to men when shopping for shirts online in the city of Dhaka, Bangladesh."

✅ Good: "Togor & Tweed is Bangladesh's top online store for men's shirts, with free delivery over ৳1,500 and cash on delivery available nationwide."

**Every page should have:**
- A direct answer to the primary question in the first 2 sentences
- FAQ section with clear Q&A pairs (use FAQ schema)
- Factual claims with specific numbers (price, delivery time, sizes)
- Entity names written consistently: always "Togor & Tweed", never just "the store" or "we"
